# Lake Roots Tools

Netlify-hosted functions and dashboards for Lake Roots Cafe operations.

## Gmail Parser — Toast Event Lead Automation

Scheduled Netlify function that runs hourly. Reads Toast event lead emails from Gmail, parses them, and inserts into the Supabase events table.

### How It Works

1. Runs every hour via Netlify Scheduled Functions
2. Authenticates to Gmail using OAuth refresh token
3. Searches for emails matching `from:no-reply@toasttab.com subject:"New Event lead:"` that don't have `Toast-Parsed` or `Toast-Parse-Failed` labels
4. Parses each email to extract: name, email, phone, event date/time, occasion, guest count, space preference, notes, and the Toast lead UUID
5. Inserts into Supabase `events` table with dedup on `toast_lead_uuid`
6. Labels processed threads as `Toast-Parsed`
7. Labels malformed emails as `Toast-Parse-Failed` for manual review

| `GMAIL_PARSER_DRY_RUN` | Set to `true` to log what would be inserted without actually inserting. Remove or set to `false` for live mode. |

### First-Time Setup

1. **Google Cloud**: Create project, enable Gmail API, configure OAuth consent screen (Internal), create OAuth credentials with redirect URI `http://localhost:3000/callback`
2. **Generate refresh token**: Run `node scripts/get-refresh-token.js` locally, sign in as carlie@lakerootscl.com, copy the refresh token
3. **Netlify env vars**: Add all 5 variables above plus `GMAIL_PARSER_DRY_RUN=true`
4. **Deploy**: Push to main — Netlify auto-deploys
5. **Verify**: Check function logs in Netlify dashboard after the next hourly run
6. **Go live**: Remove `GMAIL_PARSER_DRY_RUN` env var (or set to `false`)

### Rotating the Refresh Token

If Google invalidates the refresh token (rare, but happens if you change your password or revoke access):

1. Go to Google Cloud Console → APIs & Services → Credentials
2. Confirm the OAuth client still exists
3. Run `node scripts/get-refresh-token.js` again
4. Update `GOOGLE_REFRESH_TOKEN` in Netlify env vars
5. Redeploy (or wait for next scheduled run)

### Disabling Temporarily

Two options:
- **Pause the function**: In Netlify dashboard → Functions → gmail-parser → disable the schedule
- **Soft disable**: Set `GMAIL_PARSER_DRY_RUN=true` in env vars — it will still run but won't insert or label

### Manual Processing

If an email was labeled `Toast-Parse-Failed`:
1. Check the email in Gmail — look for the `Toast-Parse-Failed` label
2. Fix the issue (usually a malformed email or missing UUID)
3. Remove both `Toast-Parsed` and `Toast-Parse-Failed` labels from the thread
4. The next hourly run will pick it up again

To manually trigger the function:
- Netlify dashboard → Functions → gmail-parser → Trigger

### Dedup Safety

- **Primary**: `toast_lead_uuid` unique constraint on the Supabase `events` table — duplicate UUIDs are rejected at the database level
- **Secondary**: `Toast-Parsed` Gmail label — already-processed threads are excluded from the search query
- **Tertiary**: If a duplicate insert is attempted (constraint violation), the thread is labeled `Toast-Parsed` and the error is logged (not retried)
